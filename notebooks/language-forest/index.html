🏔️ Chapter 2: The River Network

Rain falls on the mountain peaks. Water flows downhill through a network of streams and gates, combining and splitting, until it reaches the ocean. Each gate controls how much water passes through. Your mission: adjust the gates to control the ocean level — but you can only see the final result. You must trace the water backward to figure out which gate to change.

🎯 Mission: Hit the Target Ocean Level

Each gate multiplies the water flowing through it. The final ocean level is the result of water flowing through all gates.
🎯 Adjust the gates to make the ocean level match the target. Watch how changes propagate!
💡 Aha! You just did Backpropagation!

In real neural networks:
• Rain = input data flowing through the network
• Gates = model parameters (weights)
• Ocean level = the model's output (and loss)
• Tracing backward = computing gradients (∂loss/∂weight)

The chain rule is exactly like tracing water upstream: the effect at each gate depends on everything downstream of it. This is what the Value.backward() function does automatically!

🌳 Chapter 3: The Whispering Trees

The ancient trees of the forest communicate through an underground mycelium network — like nature's internet. When a young sapling needs wisdom, it sends a question through its roots. Each elder tree has expertise in different topics. The sapling must decide: which elders should I listen to most?

🎯 Mission: Help the Sapling Find Wisdom

The sapling has a question (Query). Each elder tree has a topic (Key) and advice (Value).
• Trees whose topic matches the question should get more attention
• Drag the attention sliders to decide how much to listen to each elder
• The sapling's wisdom = weighted mix of all elders' advice
💡 Aha! You just computed Attention!

In real transformers:
• The sapling's question = Query (Q)
• Each elder's topic = Key (K)
• Each elder's advice = Value (V)
• Matching question to topic = Q·K dot product
• Attention weights = softmax of those scores
• Final wisdom = weighted sum of Values

Multi-head attention = multiple saplings asking different types of questions simultaneously, then combining their answers!
